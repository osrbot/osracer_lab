## 当前必须测量

优先填 [参数填写表](real_car_parameter_fill_sheet.md)。

| 优先级 | 参数 | 为什么重要 |
|---:|---|---|
| 1 | 整车质量 | 影响动力学、摩擦、制动 |
| 2 | 前/后轴重 | 影响转向和抓地 |
| 3 | 前轮距 | Ackermann 几何需要 |
| 4 | 负载轮半径 | 当前 `0.0425 m` 和 `0.050 m` 未统一 |
| 5 | 左/右最大转向角 | 决定策略动作边界 |
| 6 | 转向响应时间 | 决定控制延迟和 sim2real 随机化 |
| 7 | 电池 S 数和电压 | 影响速度、保护阈值 |
| 8 | 最高速度和最低稳定速度 | 决定实车安全限速 |
| 9 | 相机内参 | 视觉 sim2real 必需 |
| 10 | 相机/雷达/IMU 外参 | 当前 URDF 与 static TF 冲突 |
| 11 | 串口命令延迟 | 闭环控制必需 |
| 12 | 时间戳同步方式 | replay 和多传感器融合必需 |

## 轮径冲突

当前存在两个值：

| 来源 | 轮径 |
|---|---:|
| `osrcore` 编码器里程计 | `0.0425 m` |
| `osracer_sim` 轻量仿真 | `0.0425 m` |
| `osracer_lab` 当前 Isaac/URDF 模型 | `0.050 m` |

不要凭感觉改。请测量车辆落地承重后的轮半径，再决定：

- 编码器里程计使用哪个值
- 碰撞/视觉模型使用哪个值
- 两者是否需要分开记录

## 外参冲突

当前 `base_link -> camera_link`、`base_link -> laser`、`base_link -> imu_link` 在 URDF 和 static TF 中不一致。实车部署前必须测量并选择一个来源。
